# v0.2.0 M0 API Inventory

This note captures the first v0.2.0 M0 scaffold landed in the operator.

## Goals

- Establish the alpha API surface for v0.2.0 resources.
- Make authorization mode explicit before M1 implementation begins.
- Register scaffold controllers so new resources reconcile and report status.
- Generate CRDs early so downstream controller and CLI work can build on stable resource names.

## New API Surface

### Security and authorization

- `SecurityProfile.spec.authorization.mode`: explicit `Local` or `Ranger` selection.
- `SecurityProfile.spec.authorization.ranger`: Ranger admin URL, service name, auth secret reference, and polling interval.
- `SecurityPolicy`: local authorization policy resource for dataset and named-graph access.

### Data governance and movement

- `SHACLPolicy`: reusable SHACL bundle definition for ingest validation.
- `ImportRequest`: one-shot import request.
- `ExportRequest`: one-shot export request.
- `IngestPipeline`: long-lived ingest pipeline scaffold.
- `ChangeSubscription`: RDF Delta egress subscription scaffold.

### Workload management

- `FusekiCluster.spec.autoscaling`: alpha autoscaling shape for future HPA integration.

## Authorization model

- `Local` mode: operator-managed `SecurityPolicy` resources are the only supported authorization source.
- `Ranger` mode: Apache Ranger is the only supported authorization source.
- Mixed local-plus-Ranger composition is intentionally unsupported in v0.2.0.
- Ranger mode is expected to fail closed on connectivity or credential issues.

## Current scaffold behavior

- New v0.2.0 resources are registered in the manager and reconcile through scaffold controllers.
- Scaffold controllers set `Configured=False`, `Reason=Scaffolded`, and `Phase=Pending`.
- `SecurityProfile` now reports the effective authorization mode in status and validates Ranger auth secrets when Ranger mode is selected.

## Follow-on M0 work

- Add admission validation for mixed authorization mode usage and other cross-resource checks.
- Add sample manifests for the new resources.
- Decide whether Ranger integration remains on `SecurityProfile` or moves into a dedicated security attachment resource before the API is declared frozen.
